git commit - значит зафиксировать все сохранённые изменения и дать им название.
git reflog - для просмотра журнала событий (clone, checkout, commit)
git add - сохраняет результат работы на локалке, чтобы он появился в репозитории и не пропал.
git reset hard - команда, используемая для сброса текущей ветки к предыдущей фиксации, отбрасывая любые локальные изменения и модификации, внесенные в файлы.
git resert --soft - откатывает на указанный коммит назад и показывает локальные изменения
git resert defaut (мб mixed) - Указатели ссылок обновляются. Раздел проиндексированных файлов сбрасывается до состояния указанного коммита. Любые изменения, которые были отменены в разделе проиндексированных файлов, перемещаются в рабочий каталог.
git push - Сохранённые изменения пока не видны коллегам, потому что находятся в локальном репозитории. Нужно отправить коммиты на GitHub.  
git merge - команда выполняет слияние отдельных направлений разработки в единую ветку. 
git cherry pick - это процесс выбора коммита из ветки и применения его к другой 
git pull - используется для извлечения и загрузки содержимого из удаленного репозитория и немедленного обновления локального репозитория этим содержимым. 
git log - выводит хеш коммита
